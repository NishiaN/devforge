## チャプター1: 基礎概念（1:30〜10:00）

### シーン1-1: DevForge v9とは（1:30〜3:30）

```
[画面] 3分割アニメーション
  左: ユーザーが質問に答えている様子（チャット吹き出しアニメーション）
  中央: 歯車が回転する処理中アニメーション

[テロップ] 「質問に答えるだけ → 60+ファイル自動生成」
```

> DevForge v9は、チャット形式で約20の質問に答えるだけで、ソフトウェア開発に必要な60以上のドキュメントを自動生成するツールです。

```
[画面] 実際のDevForge v9オンボーディング画面をスクリーンキャプチャ
[ズーム] 統計バーにフォーカス: 「56ファイル / 230+技術エントリ / 8 Pillars / 10 AIツール」
[テロップ] ↑の数字を大きく表示
```

> 画面上部の統計が示すとおり、56のファイルテンプレート、230以上の技術データベース、8つの柱、10種類のAIツール対応。これらが全て、このツールに詰まっています。

```
[画面] HTMLファイルのアイコン1つ → サイズ表示「285KB」
[比較] スマホ写真(3MB)の横に並べて「約1/10」と表示
[テロップ] 「たった1つのHTMLファイル — サーバー不要・インストール不要」
```

> そして驚くべきことに、これは全て1つのHTMLファイルに収まっています。サイズはわずか285キロバイト。サーバーもインストールも不要。ブラウザで開くだけです。

### シーン1-3: 内部アーキテクチャ（7:30〜10:00）

```
[画面] モジュール構成図（ツリーダイアグラム）
┌ src/
├─ core/     （5モジュール）状態管理・i18n・イベント・ツアー・初期化
├─ data/     （5モジュール）プリセット26種・質問24問・TechDB 231件
├─ generators/（7モジュール）7つの生成エンジン
└─ ui/       （19モジュール）19のUI機能

[アニメーション] 各ディレクトリが順番に展開してモジュールが表示される
[テロップ] 「36モジュール → ビルド → 1 HTMLファイル」
```

> 内部は36のJavaScriptモジュールで構成されています。core（コア）5つ、data（データ）5つ、generators（生成エンジン）7つ、ui（ユーザーインターフェース）19つ。これらがビルドスクリプトで結合され、1つのHTMLファイルになります。

```
[画面] ビルド構成比の円グラフ
  JavaScript 81.6% (209KB) — 青
  CSS 15.2% (39KB) — 緑
  HTML 3.3% (8KB) — オレンジ
[テロップ] 「合計 285KB（gzip 88KB）」
```

> 内訳はJavaScriptが約82パーセントで209キロバイト、CSSが15パーセントで39キロバイト、HTMLが3パーセントで8キロバイトです。gzip圧縮すると88キロバイトになります。

### シーン2-1: 動作環境と起動（10:00〜12:00）

```
[テロップ] 「CDN読み込み失敗時もコア機能は動作（Progressive Enhancement）」
[アニメーション] ×印でCDN接続が切れる → コア機能が動作し続ける
```

> 補足として、ZIPエクスポートとMarkdownプレビューはCDNからライブラリを読み込みます。ただし、これらが失敗してもコア機能は全て動作します。
