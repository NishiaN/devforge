# DevForge v9 プレゼンスクリプト

> **バージョン**: v9.6.0（2026-02-26 更新）
> **対象**: デモ発表・説明会・動画台本の素材として
> **想定尺**: 15〜20分（詳細版）／ 5〜7分（ショート版）

## オープニング（1分）

「質問に答えるだけで、ソフトウェア開発に必要なドキュメントを全て自動生成できたら？」

今日ご紹介する **DevForge v9** は、まさにそれを実現するツールです。

そして特筆すべきは、これが **1つのHTMLファイル** だということ。
サーバー不要。APIキー不要。インターネット接続すら不要。
ブラウザで開くだけで、すぐに使い始められます。

---

## 1. DevForge v9 とは何か（3分）

### 一言で言うと

> **「AI駆動開発の出発点を、10分で整える統合ドキュメント生成プラットフォーム」**

チャット形式のウィザードで25問に答えるだけで、**178以上のファイル**を一括生成します。

具体的には：

- ソフトウェア設計書（.spec/）
- Docker開発環境（DevContainer）
- MCP設定ファイル
- 全AIツール向けルール（Claude / Cursor / Windsurf / Copilot / Cline / Aider / Devin / v0 / bolt / Zed）
- 技術ロードマップ（9段階）
- コンプライアンスチェックリスト（HIPAA / PCI DSS / ISMAP / APPI / OWASP）
- 102本のドキュメント（プロジェクト概要〜パフォーマンス監視）

これらが **わずか10〜15分の対話** で揃います。

## 2. 主要機能と25本の柱（5分）

### ウィザードシステム

スキルレベル（Beginner / Intermediate / Professional）を選ぶと、表示される選択肢が変わります。初心者には迷わない2択を。上級者にはAngular、Remix、Spring Boot、FastAPIまで全選択肢を。

25問への回答後、**プロジェクト複雑度スコア**（0〜100）が自動算出され、推奨チーム規模・開発期間も提示されます。

### 329種のプリセット

**63標準プリセット**でよく使われる業種・用途はワンクリックで設定完了。

- SaaS / ECサイト / SNS / 教育プラットフォーム / Fintech / 医療 / 政府ポータル / 保険 / 法務文書管理 / 農業 / 製造業 / エネルギー管理 … など

**266フィールドプリセット**では、より専門的な用途に対応：

- バイオ創薬 / 電力グリッド管理 / 太陽光モニタリング / 災害情報システム / 保険金請求 … など

### 整合性チェッカー（118ルール）

プリセット選択・ウィザード回答のたびに、118の整合性ルールがリアルタイムでチェック。

例：
- NestJS + Vercel → WARN（Vercelはサーバーレス向き、NestJSにはRailway推奨）
- Firebase + Prisma → ERROR（FirebaseはNoSQLでPrisma非対応）
- 医療 + Firebase → WARN（HIPAA考慮からSupabase推奨）

エラーは修正しないと先に進めないガードレール設計。

### コンプライアンスマトリクス

32ドメインに応じた**規制フレームワーク自動マッピング**：

| ドメイン | 適用フレームワーク |
|---------|-----------------|
| 医療・ヘルスケア | HIPAA + APPI + OWASP ASVS Lv2 |
| 金融・Fintech | PCI DSS + APPI + OWASP ASVS Lv2 |
| 政府・行政 | ISMAP + APPI + OWASP ASVS Lv2 |
| 法務 | APPI + OWASP ASVS Lv2 + 弁護士法 |
| 保険 | APPI + OWASP ASVS Lv2 + 保険業法 |
| 全ドメイン共通 | APPI + Cookie同意管理 |

## 3. 競合比較 — 強みと弱み（5分）

### 比較対象

| ツール | カテゴリ | 主な用途 |
|-------|---------|---------|
| bolt.new | AIコード生成 | ブラウザ内でフルスタックアプリ生成 |
| Lovable | AIコード生成 | プロンプトからReactアプリを生成 |
| v0（Vercel） | UIコード生成 | Reactコンポーネント生成 |
| Cursor | AI搭載IDE | コーディング補助・コード生成 |
| GitHub Copilot | AIコーディング補助 | IDE内コード補完 |
| create-next-app等 | スキャフォールディング | プロジェクト雛形生成 |

---

### DevForge v9 の強み

#### ✅ 強み1: **APIキー・サーバー・課金が一切不要**

bolt.new・Lovable・v0はAPIキー（有料）または月額課金が必要。
DevForge v9は **完全無料の単一HTMLファイル**。メールに添付して配布できます。
企業のセキュリティポリシーでAPIキーを外部送信できない環境でも使えます。

#### ✅ 強み2: **完全オフライン動作**

コア機能はネット接続ゼロで動作。
（ZIPエクスポートとMarkdownプレビューは初回のみCDN読み込みが必要）
インターネット制限のある政府機関・金融機関・病院内でも使用可能。

#### ✅ 強み3: **ドキュメント品質がAI生成より高い**

bolt.new / Lovable はAIがコードを生成します。仕様書は生成しません。
DevForge v9は**178以上の構造化されたドキュメント**を生成します。
しかもテンプレートベース（1370テスト済み）のため、ランダムなAI出力より**再現性・品質が安定**。

#### ✅ 強み4: **32ドメインのコンプライアンス知識を内蔵**

医療ならHIPAA、金融ならPCI DSS、政府ならISMAP。
競合ツールは汎用的で業界固有の規制をカバーしていません。
DevForge v9は「どのドメインで作るか」を答えるだけで、**適切な規制チェックリストが自動生成**されます。

#### ✅ 強み5: **10種のAIツール向けルールを同時生成**

Claude Code用・Cursor用・Windsurf用・GitHub Copilot用・Cline用・Aider用・Devin用・v0用・bolt用・Zed用のルールファイルを**全て同時生成**。
どのAIツールを使っても、プロジェクトの仕様に沿ったコード生成が可能になります。

#### ✅ 強み6: **329種のプリセット × 32ドメイン**

汎用SaaSから政府ポータル・保険システム・農業管理・太陽光モニタリングまで、非常に専門的な用途も1クリックで設定完了。
create-next-appなどのスキャフォールディングツールはプロジェクト構造しか生成しません。

#### ✅ 強み7: **完全バイリンガル（UIと生成ドキュメントが独立）**

UIは日本語のまま、生成ドキュメントは英語で出力する、といった設定が可能。
グローバル展開を考えるチームに実用的。

#### ✅ 強み8: **MCP設定の自動生成**

Model Context Protocol（MCP）の設定ファイルを、選択した技術スタックに合わせて自動生成。
これを競合ツールはカバーしていません。

---

### DevForge v9 の弱み

#### ❌ 弱み1: **コードを生成しない**（最大の弱点）

bolt.new・Lovableは「動くアプリ」をブラウザ内で生成します。
DevForge v9は**ドキュメントと設定ファイルのみ**を生成します。
「5分で動くプロトタイプが欲しい」という用途には完全に不向きです。

> 位置づけの言い換え：DevForge v9は「AIにコードを書かせるための準備をする」ツール。bolt.newは「AIが直接コードを書く」ツール。競合ではなく補完関係とも言えます。

#### ❌ 弱み2: **ビルドサイズ上限（3500KB）が迫っている**

現在3091KB（上限3500KB）。残り409KBのみ。
機能追加に制約が生じており、大規模な新機能の追加が難しくなっています。

#### ❌ 弱み3: **クラウド同期・クロスデバイス対応なし**

全データはブラウザのlocalStorageに保存。
別のPC・ブラウザからは引き続きできません（JSONエクスポート→インポートは可能）。
bolt.new・Lovableはクラウドでプロジェクト管理できます。

#### ❌ 弱み4: **リアルタイムコラボレーション非対応**

複数人が同時に編集する機能はありません。URL共有（Base64エンコード）でスナップショットの共有は可能ですが、リアルタイム共同編集はできません。

#### ❌ 弱み5: **テンプレートベースのため適応力に限界**

LLMによる生成ではないため、「このプロジェクトならではの表現」が出にくい面があります。
ドメイン固有の深い業務知識が必要な記述は、生成後に人間が追記する必要があります。

#### ❌ 弱み6: **自動アップデートがない**

静的ファイルのため、新バージョンが出たら手動で差し替えが必要です。
SaaSツールなら自動的に最新版が使えます。

## 4. どんな人・場面に向いているか（2分）

### ✅ DevForge v9 が最も活きる場面

1. **新規プロジェクト立ち上げ時**
   技術選定〜仕様書〜AI向けルール設定を10〜15分でゼロから揃えたいとき

2. **AIコーディングツールに仕様を伝えたいとき**
   Claude Code・Cursorへの投入用プロンプトや設定ファイルをまとめて生成

3. **チーム・クライアントへの技術仕様共有**
   178ファイル分の情報をZIPで配布、または URL 共有

4. **コンプライアンス対応が必要な業種**
   医療・金融・政府・保険・法務の規制チェックリストを素早く入手

5. **API制限がある環境**
   インターネット制限がある企業・官公庁内でもオフライン動作可能

### ❌ DevForge v9 が向いていない場面

- 今すぐ動くプロトタイプが必要 → bolt.new / Lovable を使う
- UIコンポーネントだけ欲しい → v0 を使う
- コード補完が欲しい → Cursor / Copilot を使う

## 5. まとめ（1分）

DevForge v9は、AI駆動開発の**「はじめの15分」**を変えるツールです。

- **63モジュール、3091KB** — 1つのHTMLファイルに凝縮
- **329種のプリセット × 32ドメイン** — 専門業種もワンクリック
- **178以上のファイル生成** — ドキュメントは全て揃う
- **118の整合性ルール** — 技術的な矛盾を事前に検出
- **10種AIツール対応** — どのAIを使っても最適化された環境

bolt.newがAIで「コードを書く」ツールなら、
DevForge v9はAIに「コードを書かせるための設計図を作る」ツールです。

この2つを組み合わせることで、**「設計→実装」のサイクルが劇的に加速します。**
