## 2. 実業務の統合モデル（あなた基準）

あなたの業務は、分解すると**4レイヤー**です。

```
A. 企画・思想
B. 設計・判断
C. 制作・量産
D. 運用・改善
```

これを **AIスキルとして再構築**します。

------

## 3. 統合スキルマップ（全業種共通）

```
【A 企画】
 └ 教育・業務テーマ定義Skill
        ↓
【B 設計】
 ├ 要件整理Skill
 ├ 仕様レビューSkill
 ├ アーキテクチャ妥当性評価Skill
        ↓
【C 制作】
 ├ 構成設計Skill
 ├ 原稿・アウトプット生成Skill
 ├ 人間らしいリライトSkill
        ↓
【D 運用】
 ├ 品質チェックSkill
 ├ 改善提案Skill
 └ 次工程設計Skill
```

✔ すべて **1スキル＝1判断**
 ✔ 人がやるのは「最初の入力」と「最終確認」だけ

------

## 4. プロ視点のスキル設計原則（超重要）

### ❌ ダメな例

- 目的が広い
- 判断基準が曖昧
- 出力が毎回違う

### ✅ 良いスキルの条件

- 目的は1文
- 判断基準は**数値 or 段階**
- 出力形式固定
- 次スキルに渡せる
