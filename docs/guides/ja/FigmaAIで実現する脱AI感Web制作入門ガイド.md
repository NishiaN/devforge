## 2. 制作環境の構築：必要なツールとセットアップ

まずは、AIとデザインをシームレスにつなぐための「3つの神器」を整えます。

### ツールセットアップ・チェックリスト

- [ ] **Figma（デスクトップアプリ版）**：MCP（Model Context Protocol）連携に必須です。ブラウザ版ではなくアプリ版をインストールしてください。
- [ ] **Claude Code**：Figmaの構造を読み取り、忠実にコーディングするメインパートナー。
- [ ] **Google AI Studio（Gemini 3.0 Pro）**：創造的な演出や3D表現を追加する際の強力な味方。
- [ ] **VS Code & Live Server拡張機能**：書いたコードを即座にブラウザで確認するために必須です。
- [ ] **Netlify**：完成したサイトを世界に解き放つためのデプロイプラットフォーム。

### コストと設定の重要ガイド

FigmaとAIを直接対話させるには、以下の設定が必要です。

1. **課金設定**：FigmaのMCP機能を利用するには、有料プラン（月額3,000円程度）への加入が必要です。これは「プロの品質を買うための投資」だと考えてください。
2. **ローカルMCPサーバーの有効化**：
   - Figmaアプリの左上「基本設定（Preferences）」→「ローカルMCPサーバーを有効化（Enable local MCP server）」にチェックを入れます。
3. **Claude Codeとの接続**：
   - ターミナルを開き、以下のコマンドを実行してClaudeをFigmaサーバーに接続します。
   - `npx -y @modelcontextprotocol/server-figma`
   - ※これがなければ、ClaudeはあなたのFigmaの中身を覗くことができません。

### AIツールの使い分け：Claude vs Gemini 3.0 Pro

| 特徴           | Claude Code (Figma連携)           | Google AI Studio (Gemini 3.0 Pro)        |
| -------------- | --------------------------------- | ---------------------------------------- |
| **得意分野**   | デザインの構造理解・忠実な再現    | **圧倒的な創造性・3D（Three.js）表現**   |
| **強み**       | MCP経由でデザインリンクを直接解析 | 抽象的な指示から「期待を超える」UIを提案 |
| **推奨シーン** | テンプレート通りに正確に組む時    | 3Dモデル導入や驚きのある演出を加えたい時 |

道具が揃ったら、次は「デザインの原石」を探しに行きましょう。

## 4. 実践コーディング：Figmaのデザインをコードに変換する

ここからが本番です。AIと対話し、デザインを現実に変えていきます。

### Claude Codeによる忠実再現ステップ

1. **リンクの取得**：Figma上でコード化したい要素を選択し、`Ctrl + L`（または右クリックで「選択範囲へのリンクをコピー」）を実行。
2. **プロンプトの実行**：Claude Codeにリンクを貼り付け、「このデザインをTailwind CSSで忠実に、完全に再現してください」と指示します。
3. **プレビュー**：生成されたファイルをVS Codeで開き、**Live Server**を起動してリアルタイムで確認します。

**⚠️ メンターの重要警告：** コーディング中は必ず **Figmaデスクトップアプリを起動したまま** にしてください。アプリを閉じるとMCP経由の画像リンクが即座に切れてしまいます。

### Gemini 3.0 Proによる「創造的」ブラッシュアップ

デザインに「魂」を吹き込みたいなら、Gemini 3.0 Proの出番です。

- **「余白（Imagination Space）」を与える**：あえて細かく指定せず、「圧倒的なクオリティで」「限界を超えてエレガントに」といった抽象的な形容詞を投げかけてください。AIの想像力が爆発し、プロ顔負けのUIが提案されます。
- **デザイン用語でブースト**：「パララックス」「ニューモフィズム」「スクロール連動アニメーション」などの用語を指示に混ぜることで、リッチなエフェクトが実装されます。

### 60点から100点へ高める反復プロセス

最初から完璧なコードは出てきません。

1. **セクションごと**にコードを生成させ、Live Serverで確認。
2. 「余白が狭い」「このボタンをもっとプレイフルに」と微調整を依頼。
3. AIが失敗したら「自分のエラーを再分析して」と促す。 この反復こそが、あなたのクリエイティビティを磨く修行になります。

現代のWeb制作において重要なのは、静的なコードを書くことではなく、サイトの「感触」や「動き」を反復的に磨き上げる**「バイブ・コーディング（Vibe Coding）」**という概念です。あなたがクリエイティブな火種を投げ込まない限り、AIは決して「プロの壁」を超えてはくれません。

## 4. Figma × MCP：デザインを完全に掌握するプロのワークフロー

Figma上のデザインをAI（Claude Code等）に直接読み込ませることで、寸分狂わぬコード化が可能になります。

### Figma MCP（Model Context Protocol）活用手順

1. **「Dev Mode」の有効化:** Figmaデスクトップアプリの「Preferences（設定）」から「Enable Dev Mode MCP Server」を有効にします。※この機能には有料プラン（月額約3,000円）が必要ですが、制作効率を考えれば不可欠な投資です。
2. **AIとの接続確認:** ターミナルで `claude mcp list` を実行し、Figmaサーバーが正しく認識されているか確認します。
3. **「選択範囲へのリンク」を取得:** 再現したいFrameを選択し、`Cmd + L`（または右クリック）で**「選択範囲へのリンクをコピー（Copy link to selection）」**を実行します。ファイル全体のリンクではなく、特定の選択範囲に限定することでAIのノイズを減らします。
4. **実装指示:** Claude Code等にリンクを貼り付け、「このデザインをTailwind CSSを用いて完全に忠実かつ精密に再現してください」と命じます。

### 成功へのショートカット：プラグインとテンプレート

- **HTML to Design:** 既存の優れたサイト（Apple.com等）をFigmaにインポートし、それを自分流にカスタマイズしてからAIに渡す手法は、プロが密かに行っているショートカットです。
- **Figmaコミュニティの活用:** ゼロから作らず、コミュニティにある高品質な「Landing Page」テンプレートをベースにすることで、最初から高いデザインクオリティを担保できます。

### 技術選定：Tailwind CSS CDNの利点

AIに「Tailwind CSSをCDN経由で」と指定することで、1つのHTMLファイルに全てのスタイルが完結します。環境構築の手間を省き、即座にブラウザで「バイブ・コーディング」を継続できる最大の利点となります。

## 2. 現行AI生成サイトの限界分析と差別化の必要性

既存のAIコーディングツールやLLMの直接出力には、プロフェッショナルな現場では看過できない「技術的不確実性（ガチャ要素）」が存在します。

1. **デザインの画一化:** 特徴のない殺風景なUIが生成され、ブランドの差別化に失敗する。
2. **実装の不透明性:** アニメーションのイージングや物理挙動が考慮されず、ユーザーに「安っぽさ」を感じさせる。
3. **再現性の欠如:** 同じ指示でも出力が安定せず、大規模開発におけるデザイン管理が困難。

# Figma MCP × Claude Code：プロフェッショナル・フロントエンド開発指針

## 1. 開発フローの戦略的意義と全体像

現代のWeb開発において、AI生成コードは「Design Gacha（デザインガチャ）」という致命的な課題を抱えている。これは、AIにUIを丸投げすることで出力が予測不能になり、無機質な「AI感」が漂う低品質なサイトが量産される現象を指す。

本指針は、Figma MCP（Model Context Protocol）を導入し、Figmaを**「Single Source of Truth（信頼できる唯一の情報源）」**として扱うことで、このランダム性を排除する。デザインの意図をAIに直接同期させる「Deterministic UI Generation（確実なUI生成）」により、デザイナーの感性とエンジニアの合理性を高度に融合させる。目標は、プロフェッショナル級のサイト構築から公開までを、わずか30分で完結させる圧倒的な時間対効果の実現である。

## 2. 基盤環境の構築：Figma MCPとClaude Codeの連携

高精細な開発を実現するには、FigmaのデザインデータをリアルタイムでClaude Codeに供給するバイパスの構築が不可欠である。

### 2.1 Figma MCPサーバーの有効化

Figmaデスクトップアプリを使用し、以下の正確なパスでローカルMCPサーバーを起動する。

- **設定パス:** `左上メニュー > プレファレンス（Preferences） > 基本設定`
- **アクション:** 「ローカルMCPサーバーを有効化」にチェック。これにより、127.0.0.1上でSSE（Server-Sent Events）接続が待機状態となる。

### 2.2 経済的妥当性とリソース

本機能の利用にはFigmaの有料プラン（月額約3,000円）が必要である。しかし、デザインガチャによる修正工数の肥大化を「技術負債の先行回避」という観点で見れば、このコストは極めて合理的である。

### 2.3 Claude Code側の接続確認

ターミナル上でClaude Code（エンジンはClaude Opus 4.1を推奨）を起動し、以下のコマンドで接続を確認する。

- **確認コマンド:** `cloud mcp list`
- リスト内に「Figma」が表示され、SSE経由で接続が確立されていれば成功である。

**環境構築チェックリスト:**

- [ ] Figmaデスクトップアプリが起動中であるか。
- [ ] ローカルMCPサーバーが有効化されているか。
- [ ] `cloud mcp list`でFigmaツールが正常に認識されているか。

## 3. 「AI感を消す」ためのデザイン資産の選定と活用

AI特有の画一的なデザインを回避するためには、人間が制作した高品質なデザイン資産を「インプットの起点」に据える必要がある。

### 3.1 高品質ソースの調達

Figmaコミュニティから「2024/2025年最新トレンド」や、商用利用可能な無料テンプレートを積極的に活用せよ。これらは既に人間による視覚的最適化が施されており、AI感の払拭に直結する。

### 3.2 骨格クローン・ワークフロー（Case: Apple.com）

既存の優れたサイト（例：Apple公式サイト）の構造を模倣する場合、「HTML to Figma」プラグインを用いてサイトの「Skeleton（骨格）」をFigmaにインポートする。

- **利点:** 完璧な余白設定やレイアウト構造をAIの参照元にできる。
- **限界と対策:** 画像データは正常にポートされないため、構造のみを流用し、ビジュアル要素は後述のプロセスで個別管理する。

### 3.3 コンポーネント選定の最適化

「20 Fast Dev」や「shadcn/ui」のようなコードベースのライブラリは効率的だが、真の「Vibe Coding（質感の追求）」においては、Figma-firstの手法が自由度・再現性ともに優位に立つ。

## 4. 実装プロセスの標準化：Figmaリンクからコード生成へ

デザインをコードに変換する際、一度に全てを生成させる手法は精度の低下を招く。

### 4.1 リンクベースのコンテキスト共有

Figma上で対象の要素を選択し、`Ctrl + L`（選択範囲へのリンクをコピー）を実行。Claude Codeに対し、このリンクを直接提示する。

- **プロンプト例:** 「このリンクのデザインをTailwind CSSを用いて忠実かつ完全に再現してください。画像はFigmaのものを参照してください。」

### 4.2 セクション単位の逐次実装

高精度な再現には、ページ全体ではなく、ヒーローセクション、フィーチャーグリッド、フッターと「1セクション1プロンプト」で進めることが鉄則である。

### 4.3 Tailwind CSSの建築的標準化

プレーンなCSSではなく、Tailwind CSS（CDN版含む）を標準スタックとして採用する。これは、クラスベースのスタイル定義がLLMの推論ロジックと相性が良く、デザインの「確定的な再現」に適しているためである。

## 5. 高度なビジュアル・ポリッシュとインタラクション

Gemini 3.0 Pro等の最新モデルが持つ「デザインの潜在的嗜好」を引き出し、サイトをプロ級の質感へ昇華させる。

### 5.1 「3パターンの潜在的デザイン案」戦略

実装前に、AIに対して以下のステップを踏ませる「Pre-generation Analysis」を推奨する。

- **指示内容:** 「ユーザーが無意識に『洗練されている』と感じる要素を分析し、異なるデザインパターンを3つ提示せよ。その中から最適な1つを選択し、実装に移行する。」

### 5.2 メタ・プロンプティングによる質感の定義

## 6. 「Persistence Protocol」：デプロイと画像の永続化

デプロイにおける最大の技術的落とし穴は、Figma MCPのセッション切断に伴う「画像リンク切れ」である。

### 6.1 永続化プロトコルの実行（最重要）

Figma MCP経由で生成された画像リンクは、Figmaアプリを閉じると即座に無効化される。これを回避するための手順を厳守すること。

1. **Figmaエクスポート:** 必要な画像をFigmaからローカルの`/images`ディレクトリに手動でエクスポートする。
2. **パスの置換:** Claude Codeに対し、「画像リンクをローカルの相対パスに全て書き換えてください」と指示する。
3. **代替策:** 汎用画像であればUnsplash APIを用いて永続的なリンクに差し替える。

### 6.2 Netlify Dropによる即時公開

`index.html`と画像フォルダを含むディレクトリを「Netlify Drop」にドラッグ＆ドロップし、数秒でデプロイを完了させる。独自ドメインの割り当てやSSL設定も、公開後のダッシュボードから迅速に行う。

プロンプトを「ガチャ」から「確実なエンジニアリング」へと進化させる。
